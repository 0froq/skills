## 什么是 Hard Context

**Hard Context** = 相对固定的个人上下文

与每日/每周变化的计划不同，hard context 包含：
- 工作作息和节律
- 正在长期维护的项目
- 周期性约束（固定会议、课程等）
- 当前生活阶段的状态

### 存储位置

`docs/dashboard/advisor/hard.md`

（注意：此文件存储在用户仓库中，不是 skill 的一部分，因此是**可变的**）

## 何时更新

### 立即更新的触发条件

当用户明确告知以下变化时：

| 变化类型 | 示例触发语 |
|---------|-----------|
| **工作变动** | "我换工作了"、"我开始新工作了"、"我离职了" |
| **项目变更** | "我不再维护 X 项目了"、"我开始负责 Y 项目"、"X 项目暂停了" |
| **作息调整** | "我调整工作时间为..."、"我开始早起了"、"我换到晚班了" |
| **阶段变化** | "我要准备考试了"、"我开始创业了"、"我进入忙季了" |
| **环境变化** | "我搬家了"、"我开始远程工作了"、"我换办公室了" |
| **临时约束** | "我下周要出差"、"我这月要准备答辩"、"我下个月有重要截止日期" |

### 每周检查

在 `start-my-week` 时，**主动询问**：

> "这周有什么生活状态的变化需要我记录吗？比如：
> - 工作时间的调整
> - 项目状态的变化
> - 新的周期性安排（课程、会议）
> - 即将到来的出差/假期"

**注意**：不需要每天询问，每周一次即可。

## 更新流程

### 第 1 步：读取现有 hard.md

如果文件存在，先读取了解当前状态。

### 第 2 步：识别变化

从用户对话中提取需要更新的信息：

**示例对话**：
```
用户: 我这周开始新工作了，上班时间变成 10-19 点

AI: 恭喜新工作！我来更新你的固定上下文：
- 工作时间: 10:00 - 19:00（之前是 09:00-18:00）
- 高效时段可能需要重新摸索，暂时保持不变

还有其他需要同步的变化吗？比如：
- 新项目相关的投入时间？
- 是否有新的固定会议或周期性安排？
```

### 第 3 步：生成更新后的文件

基于现有内容和新的变化，生成更新后的 hard.md：

- **保留未变化的部分**
- **更新变化的部分**
- **添加更新日志条目**
- **保持整体结构一致**

### 第 4 步：确认并写入

向用户展示变更摘要：

> "我将更新 hard.md 中的以下内容：
> - 工作时间: 09:00-18:00 → 10:00-19:00
> - 更新日志: 添加 2026-04-01 工作时间调整记录
> 
> 确认更新吗？"

用户确认后，写入文件并 git 提交。

## 对话示例

### 场景 1：用户主动告知变化

```
用户: 我下周要出差 3 天

AI: 了解！我来更新你的固定上下文，记录这次出差：

> 在 hard.md 的"⚠️ 当前约束"部分添加：
> - **2026-04-08 至 2026-04-10**: 出差
>   - 影响: 工作时间减少，只能处理轻量任务
>   - 建议: 此期间安排沟通、review 类任务

还有其他需要同步的变化吗？比如这次出差会不会影响某个项目的截止日期？
```

### 场景 2：每周主动询问（start-my-week）

```
AI: 新的一周开始了！在规划之前，快速检查一下：

这周有什么生活状态的变化需要我记录吗？比如：
- 工作时间的调整
- 项目状态的变化（开始/暂停/结束某个项目）
- 新的周期性安排（课程、固定会议）
- 即将到来的出差、假期或其他重要安排

如果没有变化，我们就直接进入周计划。

用户: 我下周开始要准备答辩，持续两周

AI: 收到！这是一个重要的临时约束。我来更新：

> 在 hard.md 中添加：
> - **2026-04-15 至 2026-04-28**: 准备答辩
>   - 影响: 可用工作时间减少 50%
>   - 建议: 这两周减少任务量，只保留最高优先级事项

这两周的周计划我会帮你相应调整。
```

### 场景 3：从对话中察觉变化

```
用户: 最近那个 legacy-project 真的好烦，维护起来太耗时了

AI: 听起来这个项目占用了你不少精力。需要我更新 hard.md 中的项目状态吗？

比如：
- 将 legacy-project 从"维护模式"改为"准备交接/停止维护"
- 调整每周投入时间为 0 小时
- 添加备注说明计划逐步退出

你想怎么处理这个项目？
```

## 与 Start-My-Week 的集成

在 `start-my-week` 的对话流程中，**第一步**应该是：

1. 读取 `hard.md` 了解固定约束
2. **主动询问**："这周有什么生活状态的变化需要我记录吗？"
3. 如果有变化 → 更新 `hard.md` → 然后继续周计划
4. 如果无变化 → 直接进入周计划对话

这样可以确保：
- 每周都有一次检查机会
- 用户不需要记住要主动告知变化
- AI 也不会每天烦人地询问
